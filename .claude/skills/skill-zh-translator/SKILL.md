# Skill 汉化器（指定目录）

## 目标

将用户指定的 skill 文件夹中的可翻译文案转换为简体中文，同时严格保持原有结构和可执行内容不变。

## 何时使用

当用户提出以下类型请求时使用本技能：
- 把某个 skill 翻译成中文
- 汉化指定 `SKILL.md` 或整个 skill 包
- 将现有英文技能本地化为简体中文版本
- 保持格式不变，仅翻译说明文字

## 输入约定

用户至少提供一个 skill 目录路径，例如：
- `skills/example-skill`
- `/abs/path/to/some-skill`

若用户未指定输出方式，使用默认策略：
- 输出目录：`<原目录名>-zh`（与原目录同级）
- 不覆盖源目录

若用户明确要求“原地覆盖”，再执行覆盖写入。

## 翻译范围

默认处理以下文本文件（若存在）：
- `SKILL.md`
- `references/**/*.md`
- `agents/**/*.md`
- 其他文档类文件：`*.md`, `*.txt`

默认不翻译二进制文件与代码文件（除非用户明确要求）：
- 图片、PDF、字体、压缩包
- `*.py`, `*.js`, `*.ts`, `*.sh`, `*.json`, `*.yaml`, `*.yml`

## 执行流程

1. 校验输入路径
- 目录必须存在
- 目录内应包含 `SKILL.md`
- 若校验失败，停止并提示修复项

2. 规划输出目录
- 默认创建同级副本目录：`<skill-dir>-zh`
- 复制源目录完整结构到输出目录

3. 逐文件翻译
- 只翻译“自然语言说明文本”
- 对每个目标文件保持原始段落与结构顺序

4. 完成后校验
- 文件数量与目录结构保持一致
- 代码块、命令、链接、占位符未破坏
- Frontmatter 关键字段完整

5. 输出结果摘要
- 给出输出目录
- 给出处理文件数、跳过文件数、风险提示

## 保留规则（必须遵守）

翻译时必须保持以下内容不变：
- 目录结构和文件名
- Markdown 标题层级、列表结构、表格结构
- 代码块内容（fenced code block）
- 行内代码（反引号包裹内容）
- 命令、参数、路径、URL、环境变量名
- 占位符与模板变量，如 `%s`、`%(name)s`、`{name}`、`${VAR}`、`{{ value }}`
- Frontmatter 的键名（例如 `name`, `description`, `compatibility`）

允许翻译：
- 段落叙述、说明文字、注释性文案
- Frontmatter 中可读文本值（通常为 `description`），但 `name` 值不翻译

## 质量检查清单

输出前执行以下检查：
- `SKILL.md` frontmatter 分隔符完整（`---` 成对）
- Markdown 代码块分隔符数量成对
- 所有链接语法仍可解析（`[text](url)`）
- 示例命令可读且未被中文化破坏
- 未出现关键标识符丢失（路径、变量、占位符）

## 失败处理

若出现以下情况，立即停止并报告：
- 输入目录不存在
- 缺失 `SKILL.md`
- 翻译后结构损坏（如代码块不闭合、frontmatter 损坏）

报告中需包含：
- 失败文件路径
- 失败原因
- 可执行修复建议

## 输出模板

```markdown

- 源目录：`<source_dir>`
- 输出目录：`<output_dir>`
- 处理文件：`<translated_count>`
- 跳过文件：`<skipped_count>`
- 执行状态：`成功` / `失败`

## 说明
- <关键说明 1>
- <关键说明 2>

## 后续建议
1. <建议 1>
2. <建议 2>
```
