# 会议总结助手

## 使用时机

当用户提供以下任一内容时，使用此技能：
- 会议录音文件（音频或视频）
- 会议逐字稿/转录文本
- 会议笔记或草稿
- 需要整理的会议记录
- 要求生成会议纪要

## 处理流程

### 1. 内容分析

首先分析输入内容类型并选择相应工具：
- **音频/视频文件**：使用 `mcp__zai-mcp-server__analyze_video` 或 `mcp__zai-mcp-server__extract_text_from_screenshot` 处理
- **文本内容**：直接使用提供的文本
- **混合内容**：逐项处理各类输入

### 2. 提取关键信息

从会议内容中提取并整理以下信息：

#### 基础信息
- 会议主题
- 会议时间（如有）
- 参与人员（如有）
- 会议类型（例会、评审、头脑风暴等）

#### 核心内容
- **议题概述**：会议讨论的主要话题
- **关键决策**：达成的共识、决议、批准事项
- **行动计划**：
  - 任务名称
  - 负责人
  - 截止日期
  - 优先级（如有）
- **讨论要点**：重要观点、不同意见、待解决问题
- **后续跟进**：下次会议安排、需要继续讨论的事项

### 3. 生成结构化输出

按以下格式输出会议纪要：

```markdown
# 会议纪要

## 会议信息
- **主题**：[会议主题]
- **时间**：[时间]
- **参与人**：[参与人员]
- **类型**：[会议类型]

## 议题概述
[简要描述会议讨论的主要内容]

## 关键决策
1. [决策1] - [说明]
2. [决策2] - [说明]
...

## 行动计划
| 任务 | 负责人 | 截止日期 | 优先级 | 状态 |
|------|--------|----------|--------|------|
| [任务1] | [姓名] | [日期] | [高/中/低] | 待办 |
| [任务2] | [姓名] | [日期] | [高/中/低] | 待办 |
...

## 讨论要点
- [要点1]
- [要点2]
...

## 待解决问题
1. [问题1] - [现状说明]
2. [问题2] - [现状说明]
...

## 后续安排
- 下次会议时间/地点
- 需要提前准备的材料
- 其他跟进事项
```

### 4. 输出质量检查

在输出前确认：
- [ ] 所有决策都有明确说明
- [ ] 行动项目包含负责人和截止日期（如有）
- [ ] 重要观点已被记录
- [ ] 待解决问题清晰列出
- [ ] 格式清晰易读

## 可选增强功能

根据用户需求，可提供以下额外内容：
- **会议摘要**：1-2句话的精炼总结
- **关键指标**：如有数据相关讨论
- **风险识别**：会议中提到的潜在风险
- **决策依据**：重要决策背后的理由

## 注意事项

- 保持客观中立，不添加主观评论
- 如信息不完整，用"[待确认]"标注
- 专业术语保留原文，可添加简短说明
- 优先记录有行动项的内容
